## Overview

The `Security` class provides cryptographic functions for secure radio communication, including AES-128 encryption, HMAC authentication, device ID management, and binding phrase support.

## Class Definition

Location: `lib/Security/Security.h`

```cpp
class Security {
public:
    Security();
    
    bool begin();
    
    // Pairing key management
    bool generatePairingKey();
    bool setPairingKey(const uint8_t* key);
    bool getPairingKey(uint8_t* key);
    bool hasPairingKey();
    
    // HMAC authentication
    void calculateHMAC(const uint8_t* data, size_t dataLen, uint8_t* hmac);
    bool verifyHMAC(const uint8_t* data, size_t dataLen, const uint8_t* hmac);
    
    // Device ID management
    bool generateDeviceId();
    bool getDeviceId(uint8_t* deviceId);
    bool setPairedDeviceId(const uint8_t* deviceId);
    bool getPairedDeviceId(uint8_t* deviceId);
    bool hasPairedDeviceId();
    
    // Encryption/Decryption
    bool encrypt(const uint8_t* plaintext, size_t len, uint8_t* ciphertext, 
                 uint16_t sequence, const uint8_t* deviceId);
    bool decrypt(const uint8_t* ciphertext, size_t len, uint8_t* plaintext, 
                 uint16_t sequence, const uint8_t* deviceId);
    
    // Binding phrase UID
    bool generateBindingUID(const char* bindingPhrase);
    bool getBindingUID(uint8_t* uid);
    bool hasBindingUID();
    bool verifyBindingUID(const uint8_t* uid);
    
    // Hardware device ID (RP2040)
    bool loadDeviceIdFromSilicon();
};
```

## EEPROM Storage Layout

```cpp
#define SECURITY_MAGIC_ADDR 120        // 4 bytes magic value
#define PAIRING_KEY_ADDR 128           // 16 bytes pairing key
#define DEVICE_ID_ADDR 144             // 8 bytes device ID
#define PAIRED_DEVICE_ID_ADDR 152      // 8 bytes paired device ID
#define BINDING_UID_ADDR 160           // 8 bytes binding UID
```

## Security Features

### Pairing Process

1. Each device has a unique 8-byte device ID (from silicon or generated)
2. TX and RX share a 16-byte pairing key (exchanged during pairing)
3. Binding phrase generates an 8-byte UID for initial pairing verification
4. After pairing, devices store each other's device IDs

### Packet Authentication

- All packets include 4-byte truncated HMAC-SHA256
- HMAC computed over: device ID + channel data + sequence number
- Prevents replay attacks and unauthorized transmitters

### Encryption

- AES-128 encryption for channel data
- IV derived from sequence number + device ID
- Prevents eavesdropping on RC commands

## Complete Usage Example

From `src/tx_main_sx128x.cpp:104`:

```cpp
Security security;
uint8_t txDeviceId[DEVICE_ID_SIZE];
uint8_t pairedRxDeviceId[DEVICE_ID_SIZE];
bool hasPairedRxId = false;

void setup() {
    if (security.begin()) {
        Serial.println("Security initialized");
        
        // Get this device's ID
        security.getDeviceId(txDeviceId);
        
        // Get binding UID
        uint8_t bindingUID[BINDING_UID_SIZE];
        if (security.getBindingUID(bindingUID)) {
            Serial.print("Binding UID: ");
            for (int i = 0; i < BINDING_UID_SIZE; i++) {
                Serial.print(bindingUID[i], HEX);
            }
            Serial.println();
        }
        
        // Check if paired
        if (security.getPairedDeviceId(pairedRxDeviceId)) {
            hasPairedRxId = true;
            Serial.println("Already paired");
        }
    }
}
```

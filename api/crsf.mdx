## Overview

The `CRSF` class implements the Crossfire Serial Protocol (CRSF) used for transmitting RC channel data to flight controllers. CRSF is a high-speed, low-latency protocol commonly used in FPV drones.

## Class Definition

Location: `lib/CRSF/CRSF.h`

```cpp
class CRSF {
public:
    CRSF(HardwareSerial* ser, uint8_t tx, uint8_t rx);
    void begin();
    void sendChannels(uint16_t channels[], uint8_t numChannels);
    void sendRCChannels(uint16_t channels[], uint8_t numChannels = 8);
};
```

### sendRCChannels()

Transmits RC channel values with a default of 8 channels.

```cpp
void sendRCChannels(uint16_t channels[], uint8_t numChannels = 8);
```

<ParamField path="channels" type="uint16_t[]">
  Array of channel values (1000-2000 µs)
</ParamField>

<ParamField path="numChannels" type="uint8_t" default="8">
  Number of channels to send (defaults to 8)
</ParamField>

**Example:**


## CRC Calculation

The class includes internal CRC8 calculation for CRSF frame integrity:

```cpp
uint8_t crc8(const uint8_t* data, uint8_t len);
```

This is handled automatically by the send methods.

## Channel Mapping

Standard RC channel mapping:

| Channel | Function | Typical Range |
|---------|----------|---------------|
| CH1 | Aileron (Roll) | 1000-2000 µs |
| CH2 | Elevator (Pitch) | 1000-2000 µs |
| CH3 | Throttle | 1000-2000 µs |
| CH4 | Rudder (Yaw) | 1000-2000 µs |
| CH5-8 | Aux switches | 1000-2000 µs |

## Protocol Details

- **Baud Rate:** 420,000 bps
- **Address:** 0xEE (Flight Controller)
- **Frame Type:** 0x16 (RC Channels)
- **Max Channels:** 16
- **CRC:** 8-bit polynomial checksum
